# UI — Streamlit baseline (historical snapshot, 2026-07-11)

> **Retired.** This describes the first UI, built in Streamlit, before it was
> replaced by the Angular + FastAPI frontend (ADR-0017). The `ui/` code it refers
> to has been removed from the repo. Kept as a record of the baseline the current
> frontend was built to match and exceed.

## What it was

A Streamlit multipage app (`ui/streamlit_app.py` + `ui/views/`), three views
behind a sidebar nav, with a shared sidebar **Health** panel.

### Ask — the user's view
- A question box → a grounded answer generated by Claude Haiku from the retrieved
  passages.
- **Evidence** section: the exact source passages (file · page) the answer was
  built from — the glass-box trust mechanic.
- A per-stage **timing + cost** line, e.g.
  `⏱ 7.1s server · connect 0.0s · retrieval 4.2s · generation 2.9s · other 0.0s · $0.0030`.
- Sidebar: indexed corpus (248 + 236 chunks) and a `k` slider.

### Evals — the developer's view
- **Retrieval**: recall@k run live over the golden set (vector / hybrid), with a
  per-question hit/miss list. Current: **recall@5 = 0.60** (currency is the miss).
- **Answers**: the last saved answer-eval — **mean 84.8**, **pass rate 90%**, with
  per-question score + critique.

### Usage — the admin's view
- Query-log telemetry (JSONL): questions logged, avg latency, **total + avg cost**,
  and a recent-questions list.

### Health — shared (DB-only today)
- A sidebar panel: `DB reachable · 37 ms · checked …`. (Being expanded into a full
  multi-dependency **Health** tab — see the mockups / ADR-0017.)

## What Streamlit gave us

From a spec-search box to a **cited-answer glass box** that is measured
(recall 0.60, answers 84.8/90%) and observable (per-stage timing, cost, health) —
in a handful of sessions. It also flushed out real bugs *in the browser* that unit
tests missed: the ~10s `localhost`/IPv6 DB hang, and a cwd-relative golden-set path
that only broke under Streamlit. Its job — discovering **what** to build, cheaply —
is done.

## Why we're moving on

The design ceiling. The target UX (top nav, a search hero, inline citation chips,
a live multi-dependency Health view) needs a real frontend Streamlit can't render.
And cloud deploy (M8) needs an API layer regardless. The next step is an Angular
SPA over a FastAPI backend — see **ADR-0017**, with the mockups as its spec.
